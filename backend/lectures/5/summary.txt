## 📚 강의 주제 및 목표
이 강의는 HTTP(Hypertext Transfer Protocol)의 기본 개념과 동작 원리를 이해하고, HTTP 버전별 특징과 성능 개선 방안을 학습하는 것을 목표로 한다. HTTP 요청과 응답 과정, 그리고 지연시간과 네트워크 성능의 관계를 다룬다.

## 🎯 핵심 개념
1) HTTP 프로토콜 개념 및 HTTP 메시지 구조 (요청, 응답)
2) HTTP 버전별 특징: HTTP/1.0, HTTP/1.1, HTTP/2
3) 네트워크 지연시간이 웹 성능에 미치는 영향과 HTTP 성능 최적화 기술

## 📖 주요 내용
HTTP는 웹 페이지와 같은 객체를 전송하기 위해 클라이언트와 서버 간에 메시지를 주고받는 프로토콜이다. 웹 페이지는 HTML, 이미지, 영상 등 다양한 객체의 집합이며, URL로 식별된다. HTTP 메시지는 클라이언트의 요청(GET, POST 등)과 서버의 응답(상태 코드와 데이터)으로 구성된다.

TCP 소켓 위에서 HTTP가 동작하며, 클라이언트는 HTTP GET 요청을 보내고 서버는 이에 대한 HTTP 응답을 반환한다. 응답에는 상태 라인, 헤더, 데이터가 포함되며, 흔히 200 OK 같은 상태 코드가 사용된다. HTTP 프로토콜은 HTTP/1.0부터 시작해 HTTP/1.1, HTTP/2로 발전하며, 각 버전은 성능 개선(지연시간 감소, 지속 연결, 파이프라이닝, 멀티플렉싱 등)을 목표로 한다.

특히, 웹 성능에서 네트워크 지연시간이 페이지 로딩 시간에 큰 영향을 주며, 사용자 경험에 직결된다. HTTP/1.1은 지속 연결과 파이프라이닝을 도입해 지연시간을 줄였고, HTTP/2는 멀티플렉싱과 서버 푸시, 헤더 압축 등으로 더 효율적인 데이터 전송을 가능하게 한다.
